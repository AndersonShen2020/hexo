使用 Firebase 建立圖床服務

為何不放在 Github 空間就好？

因為 Github 有空間限制，好像大概 1GB。

Firebase 的流量

提供空間 5 GB，1 天的下載量有 1 GB。

如何使用 Firebase

新增專案

進入 Firebase 官網 ，點擊 Go to console 。

點選 建立專案 。

輸入專案名稱。

要不要啟用 GA 分析。

可以新增帳戶，或是啟用以往建立的帳戶。

設定地區。

建立中。

建立完後繼續。

轉跳到 Firebase 後台。

建立 Storage 服務

找出左邊選單中的 建構 > Storage 。

按下開始使用。

選擇正式版。

只是用來儲存圖片，所以位置應該是沒差？

建立過程大約需要 1 ~ 2 分鐘。

建立完成後會轉跳到 Storage 服務畫面。

使用 Storage 服務

上傳圖片，這樣就可以 Storage 中看到上傳的內容。

複製連結網址，這樣就可以取到這張圖片。
